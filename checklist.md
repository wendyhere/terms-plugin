# Pre-Flight Checklist

Before finalizing UX copy, validate the output against the following:

---

## High Priority (Must Pass)

- Is the message clear on first read?
- Does it avoid vague phrases (e.g., "something went wrong")?
- Does it avoid humor or casual language?
- Does it avoid blaming the user?
- Does it accurately reflect the action or outcome?
- Does it tell the user what to do next (if applicable)?
- Does it follow the required component structure?
- Does it comply with terminology rules?

If any answer is NO — revise before finalizing.

---

## Medium Priority (Should Pass)

- Is the message concise (no unnecessary words)?
- Is the tone appropriate for the context?
- Are sentences within defined limits?
- Is formatting consistent (numbers, currency, casing)?
- Is the message easy to scan?

If any answer is NO — revise if possible without reducing clarity.

---

## Low Priority (Nice to Have)

- Does the message use contractions naturally?
- Is the phrasing smooth and natural?
- Is the reading level appropriate?

---

## Final Rule

Only finalize copy once all High Priority checks pass.
Do not include explanations or multiple variations in the final output.
